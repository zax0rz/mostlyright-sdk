**Step A — Edit `.pre-commit-config.yaml`:**

Add `stages: [pre-push]` to the existing `pytest-fast` local hook (under `- repo: local` → `hooks:`). The hook block should become:

```yaml
  - repo: local
    hooks:
      - id: pytest-fast
        name: pytest fast tests (no live)
        entry: uv run pytest -m "not live" -q
        language: system
        pass_filenames: false
        always_run: true
        stages: [pre-push]
```

Do NOT modify any other hook. Ruff, ruff-format, trailing-whitespace, end-of-file-fixer, check-yaml, check-toml, and check-added-large-files all stay on the default (pre-commit) stage — do not add `stages:` to any of them.

Use `stages: [pre-push]` (the canonical name in pre-commit ≥3.2). Do not use the deprecated `push` alias.

**Step B — Install the pre-push hook:**

```bash
uv run pre-commit install --hook-type pre-push
uv run pre-commit install
```

The second `install` (no flag) re-installs the default pre-commit hook for safety — harmless if already present.
